# Rhapsody · 狂想曲

Rhapsody 是一个面向 SillyTavern 的高级视觉扩展。

它不改变 SillyTavern 的数据、接口、事件与核心 DOM 结构，而是在原生界面之上建立一套完整、可关闭、可恢复的视觉语言。

> 像组织一首乐曲那样组织界面：让层级形成声部，让留白成为休止，让动效服从节奏，让每一种装饰都有出现的理由。

## 名称

**Rhapsody**，狂想曲。

它既指向古典音乐中的自由曲式，也代表创作者自己的“狂想曲”：不以复刻某一种历史风格为目标，而是在秩序与自由、克制与想象之间寻找平衡。

Rhapsody 包含两套主题：

- **Nocturne · 夜曲**：内省、深邃、安静而富有层次。
- **Aubade · 晨曲**：清醒、温暖、轻盈，如黎明逐渐照亮空间。

两者不是简单的深色模式与浅色模式，而是同一套设计语言在不同光线、温度与情绪中的两个乐章。

## 审美核心

Rhapsody 追求古典音乐与诗歌共有的美感：

- 和谐，但不单调；
- 均衡，但不僵硬对称；
- 富有节奏，但不持续制造运动；
- 保留想象空间，而不把含义全部说尽；
- 让技术隐退到体验之后。

神话元素只作为遥远、淡薄的背景隐喻存在。它可以被感受到，却不应成为表面的主题装饰。

## 项目边界

Rhapsody 将：

- 重构色彩、排版、空间、层级、材质与动效表现；
- 美化聊天、侧栏、输入区、弹窗及插件自己的独立界面；
- 提供 Nocturne 与 Aubade 双主题；
- 提供可调节、可暂停、可降级的环境效果；
- 尽可能兼容 SillyTavern 原生行为与其他扩展。

Rhapsody 不会：

- 修改聊天数据、角色数据或 SillyTavern 接口；
- 接管消息生成与核心业务逻辑；
- 为追求视觉效果大规模搬动或替换原生 DOM；
- 把界面设计成古希腊、神殿、雕塑或华丽纹章的直白主题；
- 依靠远程图床维持核心视觉。

## 当前阶段

项目已完成设计定稿、实现前技术约束与最小可运行扩展骨架。当前仓库已经包含 SillyTavern manifest、原生 ESM 生命周期、设置持久化、兼容探测、双主题令牌入口和插件设置面板；下一步从工作区 CSS 与共享动画调度器开始实现正式视觉。

后续主实现者请先阅读 [Kimi 创作与实现交接书](KIMI.md)。它说明了创作权限、不可突破的边界、本地素材库、参考项目、Figma、许可证注意事项以及无残留安装卸载契约。
