# Rhapsody 视觉基底与设计语言

> 文档状态：Draft 0.1<br>
> 文档级别：设计方向规范<br>
> 上位文档：[`DESIGN-BRIEF.md`](DESIGN-BRIEF.md)<br>
> 项目：Rhapsody · 狂想曲

本文件把《Rhapsody 设计纲领》中的审美原则转化为可以指导设计令牌、页面骨架和前端实现的视觉语言。

它回答“Rhapsody 应该看起来像什么”，但暂不锁定全部颜色数值、字号、间距和 CSS 变量。具体数值由后续 `DESIGN-TOKENS.md` 定义。

---

## 1. 核心视觉公式

Rhapsody 的视觉基底由五部分组成：

```text
macOS 的窗口秩序
        +
VS Code / Cursor 的编辑器结构
        +
适合长文本的文学排版
        +
Nocturne / Aubade 的古典音乐性格
        +
遥远而淡薄的神话隐喻
```

其中：

- macOS 提供窗口比例、表面层级与克制的系统感；
- VS Code / Cursor 提供工作区、面板、标签、状态和紧凑控件的组织方式；
- 文学排版保证角色、文字与对话仍然是视觉主声部；
- 古典音乐提供节奏、速度、重复、变奏与主题命名；
- 神话只提供远景中的历史感和想象，不成为表面装饰主题。

## 2. Rhapsody 不是什么

为了避免视觉方向逐渐偏移，Rhapsody 明确不是：

- macOS 界面的像素级仿制；
- VS Code、Cursor 或其他编辑器的主题皮肤；
- 到处显示 `.ts`、compiler、build 和 terminal 的程序员主题；
- 依赖巨大标题、强滚动劫持和持续动效的 Awwwards 模板；
- 希腊神庙、雕塑、纹章和金色花边构成的神话主题；
- 把所有素材库效果同时堆入一个页面的技术展览；
- 通过修改 SillyTavern 接口或搬运核心 DOM 实现的界面 Fork。

编辑器是结构语言，古典音乐与诗歌是精神语言。两者都必须服务于聊天，而不能覆盖聊天。

## 3. 视觉完成度目标

Rhapsody 的目标是：

> 第一眼像一个拥有独立品牌和完整设计系统的新网站，长期使用时仍然保留 SillyTavern 熟悉、稳定的操作逻辑。

允许通过 CSS Grid、Flex、视觉顺序、定位、伪元素和插件自有图层重新组织画面；不改变关键节点的身份、父子关系与事件语义。

### 3.1 截图层面的目标

在静态截图中，Rhapsody 应与 SillyTavern 默认界面有显著差异：

- 页面拥有明确的编辑器工作区构图；
- 顶部、侧栏、聊天区和输入区属于同一设计体系；
- Nocturne 与 Aubade 能被立即区分；
- 不依赖背景图片也具有完整氛围；
- 用户不会把它误认为零散 CSS 补丁的集合。

### 3.2 使用层面的目标

在实际使用中：

- 原生按钮仍在原来的功能上下文中；
- 抽屉、消息、编辑、滑动和输入保持酒馆原有行为；
- 键盘焦点顺序不因视觉重排而失真；
- 关闭 Rhapsody 后可以恢复原界面；
- 单个效果失败不会阻止聊天。

## 4. 既有 CSS 原型的继承关系

Rhapsody 已有两份纯 CSS 视觉原型：

```text
Morning Song K.2
        ↓ 提炼与重构
Aubade

Nocturne Op.4
        ↓ 提炼与重构
Nocturne
```

原型用于确认创作者已经形成的审美偏好，不直接作为插件的生产样式入口。

### 4.1 继续保留

- 紧凑的 macOS 式标题栏比例；
- 克制圆角、细边框和清楚的表面层级；
- 编辑器式工作区、标签和状态信息；
- 适合长文本的消息正文；
- 本地 SVG 图标和离线视觉；
- Nocturne 的夜蓝、柔紫和低饱和旧金；
- Aubade 的象牙白、雾蓝和晨光金；
- 对生成状态、选中状态和编辑状态的细腻反馈；
- 将动画速度与音乐节奏联系起来的思考方式。

### 4.2 降低或移除

- 过量出现的 TypeScript、编译器和构建文案；
- 为每一个区域强行添加文件名隐喻；
- 同一视口中同时发光的多个焦点；
- 每条消息都拥有过重边框、轨道和装饰的倾向；
- 为覆盖优先级而大量累积的 `!important`；
- 两套主题分别复制并逐渐分叉的巨型 CSS 文件；
- 只为桌面宽屏成立的固定尺寸和绝对定位。

### 4.3 重新建立

- 共享的语义设计令牌；
- 清楚的 CSS `@layer` 顺序；
- 统一的 SillyTavern 选择器适配层；
- 桌面、窄屏和移动端响应式策略；
- 动效档位、减少动态效果与静态回退；
- 插件启用、禁用和卸载后的完整清理机制。

## 5. 共享骨架与主题特化

Nocturne 与 Aubade 的设计比例暂定为：

```text
70% 共享编辑器骨架
20% 双主题设计令牌
10% 主题专属隐喻与动效
```

### 5.1 共享部分

两套主题共享：

- 页面信息架构；
- 窗口外壳与标题栏；
- 面板、标签、控件和弹窗结构；
- 主聊天区宽度逻辑；
- 消息、头像和操作区布局；
- 输入区域结构；
- 字号、间距和圆角比例；
- 设置中心的信息结构；
- 加载状态机与动画结构；
- 可访问性和性能降级策略。

### 5.2 主题专属部分

主题可以改变：

- 背景明度与色温；
- 表面透明度和阴影重量；
- 主强调色与次强调色；
- 环境光的位置、速度与扩散方式；
- 少量欢迎文案和状态命名；
- 加载碎片的颜色与最终收束光线；
- 空白欢迎区的意象。

主题切换不改变功能位置，不重新创建聊天内容，也不更换一套完全不同的组件。

## 6. 编辑器结构的使用方式

### 6.1 借用编辑器的部分

- 顶部窗口栏与工作区边界；
- 左右工具区与中心内容区的空间秩序；
- 紧凑、对齐明确的控件；
- 标签、状态点、路径和轻量元信息；
- 代码编辑器式的精确间距；
- 对选中、悬停、禁用、生成中和错误状态的明确区分；
- 主内容保持安静，工具在需要时出现。

### 6.2 不照搬编辑器的部分

- 不为所有区域创造虚构文件名；
- 不把用户必须理解的功能换成晦涩技术术语；
- 不把侧栏设计成真实文件树的复制品；
- 不默认展示终端、编译日志和代码行号；
- 不使用过量语法高亮颜色污染聊天正文；
- 不让编辑器隐喻阻碍非开发者使用。

### 6.3 推荐浓度

编辑器感应主要来自比例、材质、密度与状态反馈，而不是文字模仿。

```text
结构表达：强
控件表达：中等
命名表达：弱
程序员梗：极少或没有
```

## 7. 页面构图

### 7.1 外部环境

页面背景是最远的环境层，只负责建立光线和空间：

- 使用多层 CSS 渐变、低强度纹理和至多一个程序化背景；
- 不直接承载功能信息；
- 环境效果必须设置为不拦截输入；
- 聊天密度提高时，背景对比度应自动后退；
- 低性能模式下可以完全变成静态渐变。

### 7.2 窗口外壳

Rhapsody 将酒馆主界面表现为一个完整工作区，而不是多个互不相关的悬浮卡片。

窗口外壳应具备：

- 清楚但不过度厚重的边界；
- 克制的 macOS 式顶部秩序；
- 统一表面层级；
- 轻微纵深和环境分离；
- 在窄屏下自然退化为全屏界面。

红黄绿窗口点是否保留、抽象化或替换为 Rhapsody 状态点，留待页面骨架阶段确认。

### 7.3 顶部工具区

顶部区域属于“伴奏”：

- 高度紧凑；
- 图标统一使用本地 SVG；
- 普通状态低对比，悬停与选中后才清晰；
- 不用持续发光提示所有可点击元素；
- 功能分组比单个按钮装饰更重要。

### 7.4 左右侧栏

侧栏表现为编辑器工具区或检查器，而不是独立网页：

- 视觉上属于主窗口；
- 展开时与聊天区保持明确边界；
- 收起时保留轻量位置提示；
- 不通过 JavaScript 搬动原生抽屉内容；
- 第三方扩展插入的设置项必须仍然可读、可滚动。

### 7.5 主聊天工作区

聊天内容是“主声部”。

- 正文拥有最高可读性和最稳定背景；
- 弱化传统社交软件式彩色气泡；
- AI 与用户通过结构、轨道、头像、微弱色温或边注区分；
- 长回复应接近阅读稿件，而不是被装入过多卡片；
- Markdown 标题、引用、代码、列表和表格属于统一排版系统；
- 消息操作在悬停、选中或键盘聚焦时出现。

AI 消息与用户消息的最终版式仍需在页面骨架阶段确定。

### 7.6 输入区域

输入区是“作曲台”，但不需要写满音乐术语。

- 保留编辑器式精确边界和紧凑按钮；
- 输入框仍然明显是可编辑区域；
- 发送、停止、附件和扩展按钮的优先级清楚；
- 多行输入时平滑扩展，不遮挡主要内容；
- 聚焦状态可以拥有一次轻微扫光或边界变化；
- 日常输入期间不持续循环动画。

### 7.7 弹窗、菜单与通知

它们属于临时前景声部：

- 必须比背景和主工作区更清楚；
- 使用同一套表面、边框和阴影体系；
- 动画短而明确；
- 错误、警告和成功状态不只依赖颜色；
- 原生和第三方弹窗都通过适配层统一视觉，不替换业务逻辑。

## 8. 排版方向

Rhapsody 使用两类字体角色：

### 8.1 界面字体

- 用于导航、按钮、状态、设置与元信息；
- 倾向现代无衬线字体；
- 中文需保证小字号清晰；
- 西文可以接近 macOS 编辑器的中性、紧凑气质。

### 8.2 内容字体

- 用于聊天正文、引用和较长阅读；
- 可以使用更具文学感的字体，但不能牺牲屏幕阅读；
- 是否采用“正文衬线、界面无衬线”的双字体系统，留待令牌与页面测试决定；
- 代码、路径和少量状态使用等宽字体。

### 8.3 展示字体

- 仅用于 Rhapsody、Nocturne、Aubade 和加载界面的少量标题；
- 不进入普通设置项和操作按钮；
- 中文与西文展示字体需要分别校准，不能只依赖英文效果。

## 9. 主题性格

### 9.1 Nocturne · 夜曲

Nocturne 是内省的夜间编辑器工作室。

```text
基础情绪：沉静、纵深、回声、月光、内省
光线：局部、低亮度、带余韵
背景：带蓝紫倾向的深灰层次，不使用单一纯黑
强调：夜蓝与柔紫为主，旧金只作极少数温度点
材质：较深的半透明表面、细边框、克制阴影
运动：慢、低幅度、较长衰减
```

Nocturne 可以使用乐章、声部、节拍和小节的轻微隐喻，但普通操作仍使用清楚名称。

### 9.2 Aubade · 晨曲

Aubade 是黎明中的编辑器工作室。

```text
基础情绪：清澈、温暖、苏醒、空气、温度
光线：柔和扩散、逐渐显露边界
背景：象牙白、雾灰和极浅蓝的层次
强调：雾蓝为主，晨光金作为稀缺温度点
材质：轻薄表面、细线和非常柔和的阴影
运动：更轻、更短、更清醒
```

Aubade 不使用大面积纯白，也不通过大量金色制造奢华感。

### 9.3 共同身份

无论主题如何切换，用户都应认出这是同一个 Rhapsody：

- 相同的窗口比例；
- 相同的排版节奏；
- 相同的控件与面板结构；
- 相同的克制程度；
- 相同的设置入口和功能位置。

## 10. 神话隐喻的视觉位置

神话只存在于最远的环境层和极少量品牌时刻。

推荐使用：

- 难以辨认的星图弧线；
- 月相或晨曦的抽象几何；
- 像织线、羽翼或海浪但无法被明确命名的曲线；
- 极少量近似铭文比例的分隔线；
- 在静止观察后才可能发现的背景细节。

不使用：

- 具体神祇形象；
- 柱式、神殿、雕像和壁画；
- 纹章式金色装饰；
- 需要世界观解释才能理解的图标；
- 压过角色立绘和聊天内容的神话主体。

## 11. 环境效果的分工

效果必须拥有单一职责。

| 效果类型 | 推荐职责 | 禁止用法 |
| --- | --- | --- |
| CSS 渐变与纹理 | 所有页面的静态基底 | 过强噪点影响阅读 |
| Dot Field / 程序背景 | 空白欢迎区或低密度聊天背景 | 与另一套大型背景同时运行 |
| Side Rays / 柔光 | Aubade 欢迎区或特殊空状态 | 长时间照亮正文区域 |
| OGL Flowmap Portrait | 单个角色欢迎页与角色切换 | 消息列表中为每个头像创建实例 |
| Border Glow | 主题卡、焦点卡和重要状态 | 所有普通面板持续发光 |
| Fragment Assembly | Rhapsody 真实加载界面 | 与加载无关的长品牌片头 |

同一视口原则上只有一个主要动态焦点。

## 12. Rhapsody Overture：真实加载界面

### 12.1 定义

**Rhapsody Overture · 狂想曲序曲**是插件的加载界面。

它不是：

- 酒馆已经可用后强制播放的广告片头；
- 与资源状态无关的固定时长动画；
- 第二套独立启动流程；
- 修改 SillyTavern 原生加载逻辑的替代品。

它是插件自有的视觉覆盖层，用来承接 Rhapsody 的真实初始化过程：读取启动设置、应用主题令牌、挂载视觉层、连接状态监听并确认界面就绪。

### 12.2 参考来源

视觉节奏参考 `Species in Pieces` 的加载与开场机制：

- 本地参考：`D:\MySpecialFolder\in-pieces`
- 上游仓库：<https://github.com/bryanjamesinteractive/in-pieces>
- 参考提交：`d81b692dc3aca4b3e4e7f5e9f8d9635094faea05`

参考机制包括：

- 多枚 `clip-path` 碎片共同形成主体；
- 碎片以错开的延迟依次点亮；
- 无序形态逐渐收束为可辨认整体；
- 文本与图形分阶段出现；
- 加载完成后通过 class 状态进入下一画面。

Rhapsody 不复制：

- 原项目的动物轮廓；
- 原始多边形坐标；
- 图片、音频、字体与物种内容；
- 原站长时间固定编排；
- jQuery 定时器实现。

Rhapsody 使用原创碎片几何、主题颜色、加载状态机和退场方式。

### 12.3 视觉叙事

序曲表达“由碎片形成秩序”，与 Rhapsody 的设计精神一致：自由的片段最终形成和谐结构。

推荐的视觉阶段：

```text
静默背景
    ↓
少量碎片在远处显现
    ↓
碎片按节奏错峰点亮
    ↓
碎片向中心聚合，形成抽象工作区或 Rhapsody 记号
    ↓
状态确认，图形收束为一条细线
    ↓
细线扩散或淡出，显露主界面
```

最终图形不使用动物轮廓，也不应是复杂神话图像。它可以是：

- 抽象的编辑器窗口平面；
- 近似乐谱小节线的几何结构；
- 非具象的 Rhapsody 字母记号；
- 月弧与地平线共享的抽象构图。

最终图形在页面骨架与动效分镜阶段确定。

### 12.4 真实加载阶段

加载状态建议分为四个可观测阶段：

1. **Prelude / 读取**：读取本地启动镜像，确定主题和序曲开关。
2. **Theme / 着色**：应用 Nocturne 或 Aubade 令牌，避免主题闪烁。
3. **Stage / 挂载**：挂载插件自有视觉层、设置入口和必要监听。
4. **Ready / 就绪**：确认核心样式和控制器可用，执行退场。

进度来自已完成阶段或真实初始化任务权重，不使用与状态无关的假百分比。

### 12.5 开启状态

当用户开启加载动画：

- 序曲覆盖层尽早出现；
- 根据当前主题选择颜色与光线；
- 加载任务推进碎片的显现与聚合；
- 初始化完成后立即进入收束和退场；
- 动画不能为了播完完整编排而阻止已就绪的界面；
- 用户可以点击、按 `Esc` 或使用跳过按钮隐藏视觉层；
- 跳过只隐藏动画，不中断初始化任务。

### 12.6 关闭状态

当用户关闭加载动画：

- 不创建 Fragment Assembly 视觉层；
- Rhapsody 仍正常读取设置、应用主题并初始化；
- 不改变 SillyTavern 的原生加载行为；
- 不留下透明遮罩、事件监听或动画帧；
- 设置入口仍然可用，用户可以随时重新开启。

### 12.7 快速加载

当初始化速度快于完整动画：

- 不制造最低等待时间；
- 已出现的碎片可以直接进入快速收束；
- 若覆盖层尚未真正绘制，可以只执行极短淡出或完全不显示；
- 避免页面先闪现、再被加载层盖住、然后再次显现。

### 12.8 加载失败与超时

如果某个非核心视觉效果初始化失败：

- 序曲显示简短的降级状态；
- 跳过该效果并继续进入酒馆；
- 不让 Dot Field、Flowmap 或其他装饰阻止聊天；
- 记录错误供调试，但不向普通用户展示技术堆栈。

如果核心控制器无法启动：

- 移除覆盖层；
- 恢复酒馆原始可操作界面；
- 提示 Rhapsody 已安全停用。

### 12.9 主题变奏

#### Nocturne Overture

- 深蓝紫空间；
- 碎片以月光灰、夜蓝和柔紫错峰点亮；
- 旧金只用于最终确认点；
- 动作较慢、衰减较长；
- 退场像声音余韵消散。

#### Aubade Overture

- 象牙白与雾灰空间；
- 碎片以雾蓝、柔白和少量晨光金点亮；
- 光线从局部逐渐扩散；
- 动作更轻、更清醒；
- 退场像黎明使工作区逐渐清晰。

两套主题使用相同几何与状态机，只改变令牌、速度微调和光线表现。

### 12.10 动效与无障碍

- 遵循 `prefers-reduced-motion`；
- 减少动态效果时直接显示静态聚合图形与状态文字；
- 禁止快速闪烁和高对比频闪；
- 状态不能只依靠颜色表达；
- 加载文案应清楚、简短、可读；
- Canvas、计时器和监听在退场后必须销毁；
- 页面隐藏时暂停循环动画。

### 12.11 待确认事项

- 默认播放频率：每次刷新、每个浏览器会话一次，或仅首次安装／更新；
- 碎片最终形成的具体抽象图形；
- 是否显示真实百分比，或只显示阶段名称；
- 默认是否允许点击任意位置跳过；
- 完整、平衡和静谧档位的碎片数量与帧率。

这些事项由后续 `MOTION-SPEC.md` 和 `PLUGIN-SETTINGS.md` 最终确定。

## 13. 插件设置中心的视觉方向

主题、加载动画与环境效果统一放在 Rhapsody 设置中心，不散落在酒馆其他区域。

设置中心使用 macOS 偏好设置与编辑器 Settings 的混合结构：

- 左侧为紧凑分类导航；
- 右侧为清楚的设置内容；
- 普通项目使用原生表单语义；
- 主题卡可以拥有轻量实时预览；
- 开关旁提供简短结果说明；
- 高性能开销选项明确标注；
- 不使用全屏展示型菜单替代高频设置操作。

建议分类：

```text
Appearance
├── Theme
├── Interface density
└── Typography

Overture
├── Loading animation
├── Playback frequency
└── Skip behavior

Environment
├── Programmatic background
├── Pointer effects
└── Character portrait effect

Motion & Performance
├── Full
├── Balanced
└── Quiet

About
├── Version
├── Sources and licenses
└── Reset Rhapsody
```

具体字段、默认值和持久化由 `PLUGIN-SETTINGS.md` 定义。

## 14. 响应式方向

### 14.1 宽屏桌面

- 完整显示窗口外壳和多区域工作区；
- 侧栏与聊天区可以同时存在；
- 环境背景拥有最多呼吸空间；
- 仍然控制最大阅读宽度，避免正文过宽。

### 14.2 普通桌面与平板横屏

- 保留核心窗口语言；
- 侧栏按需覆盖或折叠；
- 降低环境效果面积；
- 设置中心可以由双栏转为较窄布局。

### 14.3 移动端

- 放弃桌面窗口阴影和外围留白；
- 主界面占满屏幕；
- 保留标题栏、色彩、排版与状态语言；
- 原生抽屉行为优先于编辑器构图；
- 加载碎片减少数量与运动幅度；
- 所有核心功能可触达，不以视觉完整度换取可用性。

## 15. 视觉验收标准

一个页面只有同时满足以下条件，才能被认为属于 Rhapsody：

1. 在没有环境动画时，静态排版和层级仍然成立。
2. 用户第一眼能感受到完整工作区，而不是多张无关卡片。
3. 编辑器感来自结构，不依赖大量技术术语。
4. Nocturne 与 Aubade 共享同一产品身份。
5. 至多存在一个主要动态焦点。
6. 神话隐喻不能在第一眼被明确识别。
7. 聊天文字始终是视觉主声部。
8. 关闭特效后，信息和操作仍然完整。
9. 不移动、不克隆、不替换 SillyTavern 核心 DOM。
10. 插件关闭后可以恢复原界面。
11. 核心视觉完全离线可用。
12. 长时间聊天不会因光效、动画或密度产生明显疲劳。
