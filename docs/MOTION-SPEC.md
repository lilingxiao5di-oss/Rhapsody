# Rhapsody Overture Motion Spec

> 状态：序曲动效规范 v1.0 定稿<br>
> 适用范围：序曲覆盖层，不包含日常聊天界面的微交互

## 1. 动效目标

序曲只有一条连续运动主线：**同一组 30 枚碎片从一个点形成小圆盘，再形成主题意象，最后形成产品入口。**

所有转场都应让用户感觉是同一批碎片改变坐标，而不是在不同素材之间交叉淡化。完整视觉编排建议为 8.0–8.6 秒，不包含用户停留在最终入口的时间；真实初始化较慢时，状态 01 可以延长。

## 2. 时间线

| 时间 | 状态 | 动作 |
|---|---|---|
| 0.00–0.18 s | Boot | 静默背景与起始光点出现 |
| 0.18–2.55 s | Point-to-Disc | 光点旋转；30 枚碎片错峰展开成小圆盘 |
| 2.55–3.30 s | Expansion | 圆盘放大、轻微转向并释放碎片 |
| 3.30–5.70 s | Four-Line Interlude | 四行文本依次进入，每行约 0.60 秒 |
| 5.70–6.70 s | Motif Assembly | 碎片聚合为主题固定意象 |
| 6.70–7.60 s | Theme Cadence | 意象停驻，只有极低幅呼吸 |
| 7.60–8.40 s | Shard Wordmark Assembly | 意象分解；`SILLY` 出现，30 枚碎片组成 `TAVERN` |
| 8.40 s–∞ | Wait for Click | 等待用户明确进入 |
| 点击后 0.40 s | Exit | 遮罩淡出并显露 SillyTavern |

## 3. 点到圆盘契约

- 固定使用 30 枚原创三角碎片；
- 初始视觉尺寸接近 4–8 px 光点；
- 最终圆盘为实心放射结构，不保留空心圆环；
- 桌面端最终直径建议为视口短边的 18%–23%，并设置像素上限；
- 碎片显现延迟建议为 30–40 ms；
- 圆盘整体从约 `scale(0.02)` 过渡到 `scale(1)`；
- 旋转以慢速连续变化表达加载，不使用高速 spinner；
- 起始光点在圆盘可辨认后淡出，而不是突然删除。

当前 Figma Motion 原型：3.2 秒、32 个动画节点、30 条碎片透明度轨道，并包含圆盘的缩放/旋转/透明度与光点的缩放/透明度轨道。

## 4. 真实加载门控

```text
Prelude → Theme → Stage → Ready
```

- 点到圆盘动画承接真实初始化，不显示假进度；
- 初始化未到 `Ready` 时停留在状态 01，以低幅旋转或呼吸维持反馈；
- 初始化很快时，也至少让“点形成圆盘”的动作清晰完成一次；
- 只有 `Ready` 后才允许进入 Expansion、主题意象和进入按钮；
- 非核心视觉失败时跳过该效果，核心控制器失败时立即移除覆盖层并恢复原界面。

## 5. 主题映射

| 主题 | 固定意象 | 运动差异 |
|---|---|---|
| Nocturne | Moonlit Lyre / 月下琴弦 | 更慢、余韵更长、夜蓝与柔紫、单枚旧金节拍 |
| Aubade | Aurora Fold / 曙光折幕 | 更轻、更清醒、深板岩蓝与雾蓝拉开层次、单枚香槟金节拍 |

当前不启用候选池、随机轮换或连续去重。若未来增加意象，必须先修改本规范，不能在实现层静默恢复随机逻辑。

## 6. 交互与会话

- 默认使用 `sessionStorage` 的 `rhapsody:overture:seen`，每个浏览器会话自动出现一次；
- 设置中心的“预览序曲”忽略并且不修改会话标记；
- `Enter`、`Space` 与点击进入；`Esc` 跳过；
- 最终入口必须等待真实界面可用后才激活；
- 离场时销毁 OGL/Canvas、`requestAnimationFrame`、计时器与监听器。

## 7. 减少动态效果

`prefers-reduced-motion: reduce` 下：

- 起始光点在 150–200 ms 内切换为静态小圆盘；
- 跳过旋转、错峰和四句逐词动画；
- 使用静态主题意象短暂停留后直接显示 `SILLY / TAVERN` 碎片字标与进入按钮；
- 不使用闪烁、高对比频闪或依赖颜色的状态表达。

## 8. 技术边界

序曲优先使用 CSS、SVG 与 OGL。它不要求完整 Three.js 场景，也不要求 React 运行时。React 只在未来设置中心确有复杂状态组件时再局部引入。
