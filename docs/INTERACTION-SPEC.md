# Rhapsody 日常交互与动效规范

> 状态：Approved 1.0<br>
> 定稿日期：2026-07-22<br>
> 适用范围：进入 SillyTavern 后的桌面工作区、聊天、欢迎页、原生抽屉与 Rhapsody 设置中心<br>
> 不包含：序曲分镜；序曲以 `MOTION-SPEC.md` 为准

## 1. 目标

Rhapsody 的交互应像演奏中准确而自然的触键：用户首先感到响应及时，其次才意识到动画精致。

本规范追求：

- 操作反馈在下一帧出现；
- 结构变化具有连续、可理解的空间方向；
- 动画跟随显示器刷新率，不人为降到 30 fps；
- 同一时刻只有一个主要动态焦点；
- 普通聊天保持安静，欢迎与章节时刻才允许更强的叙事性；
- 不延迟 SillyTavern 原生业务，不改变原生事件语义；
- Nocturne 与 Aubade 使用相同交互，只改变颜色、光感与少量环境参数。

## 2. 平台范围

Rhapsody 只面向桌面浏览器。

- 推荐 CSS 视口：`1440 × 800` 及以上。
- 最低设计验证视口：`1280 × 720`。
- 支持鼠标、触控板和键盘。
- 不设计手机、平板竖屏、触摸优先或小于 `1280px` 的工作区交互。
- 低于最低视口时，不启用实验性工作区布局；SillyTavern 原生功能仍不得被阻断。

这里的桌面限定不取消键盘、焦点、减少动态效果和语义标记等无障碍要求。

## 3. 交互原则

### 3.1 响应先于装饰

按下、聚焦、选中和错误反馈必须立即出现。动画可以完善反馈，但不能成为反馈本身。

```text
输入发生
→ 下一帧显示状态
→ 动画完成空间或材质过渡
→ 稳定停留
```

禁止等待动画结束后才执行发送、关闭、切换主题或保存设置。

### 3.2 空间连续

- 抽屉从其触发位置一侧进入。
- Popup 在自身最终位置轻微进入，不从屏幕远处飞入。
- 消息沿阅读方向出现，只允许极小的纵向位移。
- 主题切换不改变布局，不重新创建消息 DOM。
- 环境视差跟随指针目标值平滑趋近，不直接复制指针坐标。

### 3.3 动静分工

| 层级 | 职责 | 强度 |
|---|---|---|
| 功能反馈 | Hover、press、focus、开关、错误 | 快、明确 |
| 结构过渡 | 抽屉、popup、设置分区、主题切换 | 中等、连续 |
| 叙事时刻 | 欢迎、角色开场、章节过渡 | 低频、完整 |
| 环境运动 | 光场、尘埃、Dot Field、视差 | 缓慢、低对比 |

叙事动画播放时，环境层降低局部对比；结构动画进行时，不同时启动另一个结构动画。

### 3.4 可中断

- 用户反向操作时，动画从当前视觉状态自然转向，不先跳回起点。
- 重复点击主题切换时取消旧过渡，以最后一次输入为准。
- 抽屉和 popup 的关闭优先于离场表演。
- 页面失焦、扩展禁用或节点卸载时立即清理动画。

## 4. 时长与缓动

### 4.1 日常时长

| 名称 | 时长 | 适用场景 |
|---|---:|---|
| Press | `80ms` | 按钮按下、图标压缩 |
| Fast | `125ms` | Hover、focus、颜色反馈 |
| Standard | `200ms` | 开关、消息进入、提示状态 |
| Structure | `280ms` | 抽屉、popover、设置分区 |
| Slow | `360ms` | 主面板、主题表面切换 |
| Theme | `420ms` | Nocturne / Aubade 完整换色 |
| Poetic | `640–800ms` | 欢迎、角色开场、章节过渡 |
| Ambient | `800–1600ms` | 环境参数缓慢趋近 |

`Poetic` 和 `Ambient` 不是控件时长。普通按钮、输入和设置不能为了“高级感”使用慢动作。

### 4.2 缓动

```css
body.rhapsody-enabled {
  --rhapsody-ease-standard: cubic-bezier(0.2, 0, 0, 1);
  --rhapsody-ease-enter: cubic-bezier(0.16, 1, 0.3, 1);
  --rhapsody-ease-exit: cubic-bezier(0.4, 0, 1, 1);
  --rhapsody-ease-ambient: cubic-bezier(0.37, 0, 0.63, 1);
}
```

- 进入：`ease-enter`，前段迅速建立可见状态，后段柔和停稳。
- 离开：`ease-exit`，快速释放空间。
- 状态颜色：`ease-standard`。
- 环境：使用基于 `deltaTime` 的阻尼插值，不依赖固定帧数。
- 禁止弹性过冲用于输入框、正文、抽屉和 popup。

## 5. 运行时状态

Rhapsody 只在插件拥有的属性中表达视觉状态：

```html
<body
  class="rhapsody-enabled"
  data-rhapsody-theme="nocturne"
  data-rhapsody-motion="full"
  data-rhapsody-environment="on"
>
```

允许的计算状态：

- `data-rhapsody-theme="nocturne|aubade"`
- `data-rhapsody-motion="full|reduced|off"`
- `data-rhapsody-environment="on|off|fallback"`
- `data-rhapsody-theme-transitioning="true"`，仅在过渡期间存在
- `data-rhapsody-compat="ok|degraded"`

`system` 是保存设置时的用户偏好，不是最终 CSS 状态。控制器需要结合 `prefers-reduced-motion` 计算为 `full` 或 `reduced`。

## 6. 通用控件

### 6.1 按钮

#### Default

- 文本与图标清晰，但不持续发光。
- 主按钮使用主强调色；同一区域最多一个主按钮。
- Ghost 按钮依靠文字、图标和轻边界存在。

#### Hover

- `125ms` 内提升背景或边界对比。
- 主按钮可 `translateY(-1px)`；普通工具按钮不移动。
- 图标不单独旋转或跳动，除非图标语义就是刷新、展开或加载。

#### Active

- 下一帧进入按压状态。
- 允许 `scale(.98)` 或 `translateY(0)`，时长 `80ms`。
- 不改变按钮宽高，不引起相邻控件位移。

#### Focus-visible

- 使用 1px 实线轮廓加低强度 focus glow。
- 轮廓与背景必须具有足够区分度。
- 鼠标点击后不强制保留键盘式焦点光环；键盘导航必须显示。

#### Disabled

- 无位移、无光效、无 hover。
- 保留可辨识文本；不能只依赖降低透明度表达原因。
- 若禁用原因不明显，使用原生 title、说明文本或可访问描述。

### 6.2 图标按钮

- 桌面视觉尺寸可以紧凑，但点击区域不得小于 `32 × 32px`。
- 图标与文字按钮共享 focus 规范。
- 无可见文本的按钮必须保留原生 `title` 或 `aria-label`。
- 不替换 SillyTavern 原生图标的业务含义。

### 6.3 开关

```text
点击 / Space
→ 立即移动 thumb 并更新状态文本
→ 200ms 完成轨道颜色过渡
→ 异步保存设置
```

- 保存设置不能阻塞 thumb 移动。
- On / Off 不能只用颜色表示。
- 程序回滚时，从当前状态过渡到真实值，并显示简短错误提示。

### 6.4 输入、选择与滑杆

- Focus 在 `125ms` 内增强边界和轻光晕。
- 输入内容与光标响应不加延迟。
- Textarea 自适应高度不做补间动画，避免输入时产生滞后。
- Select 保留原生键盘行为。
- Range 滑杆的数值显示与 thumb 同帧更新；昂贵效果只在 `change` 后提交，预览使用节流后的轻量更新。

## 7. 顶栏与工具区

- 顶栏是稳定参照物，不因聊天滚动产生位移。
- 普通工具按钮 hover 只改变背景、边界或图标明度。
- 当前选中工具使用持续但静态的选中标记，不使用呼吸光。
- 多个状态同时存在时，优先级为：错误 > 生成中 > 选中 > hover。
- Rhapsody 不隐藏原生工具按钮来获得更干净的截图。

工作区布局实验若把原生 `#top-bar` 视觉化为侧栏，交互语义、焦点顺序和 drawer 触发方式仍保持原样。

## 8. 抽屉、Popover 与 Popup

### 8.1 抽屉

- 进入时长 `280ms`，位移 `8–16px`，透明度从 `.92` 到 `1`。
- 方向与抽屉来源一致。
- 遮罩在 `125ms` 内先建立，再完成面板进入。
- 关闭时长 `160–220ms`；如果原生逻辑立即隐藏，Rhapsody 不延迟业务关闭。
- 不对宽度、`left`、`right` 做逐帧动画，优先使用 `transform`。

### 8.2 Popover / 菜单

- `160–200ms`，位移不超过 `6px`。
- Transform origin 靠近触发器。
- 首次可交互时间不能等待动画结束。
- 点击外部、Escape 和原生关闭按钮立即生效。

### 8.3 Modal

- 遮罩与内容分开过渡。
- 内容使用轻微 `translateY(8px)` 与 opacity，不缩放整段文字。
- 打开后焦点进入第一个合理控件或标题；关闭后回到触发器。
- Rhapsody 不修改 SillyTavern popup 的确认与取消语义。

## 9. 欢迎页

欢迎页允许使用 B 方案的诗性排版，但动作入口必须清楚。

建议进入序列：

```text
0ms       主标题与背景光场开始建立
80ms      简短说明进入
160ms     主动作与快捷入口进入
240ms     最近会话列表整体进入
640–800ms 全部稳定
```

- 最近会话列表作为一个整体进入，不逐条长时间错峰。
- 鼠标经过最近会话时只提升当前条目，不推动其他条目。
- 原生会话操作在 hover 和 `focus-within` 时清晰可见。
- 欢迎页已有内容时不播放序曲式碎片聚合。
- 从聊天返回欢迎页可以使用更短的 `360ms` 版本，避免重复表演。

## 10. 消息区

### 10.1 已有历史

- 打开聊天时，历史消息不逐条播放进入动画。
- 首屏可以整体从 `.98` opacity 过渡到 `1`，最长 `180ms`；若会造成闪烁则完全取消。
- 加载更多历史消息时保持滚动锚点，不做位移动画。

### 10.2 新消息

新消息只允许一次轻量进入：

```css
@keyframes rhapsody-message-enter {
  from {
    opacity: 0.88;
    transform: translate3d(0, 6px, 0);
  }
  to {
    opacity: 1;
    transform: translate3d(0, 0, 0);
  }
}
```

- 时长 `200ms`，使用 `ease-enter`。
- 用户消息发送后立即占位，不等待网络。
- 角色消息在原生 `CHARACTER_MESSAGE_RENDERED` 后进入。
- 动画结束后移除临时 class 和 `will-change`。

### 10.3 流式生成

- 不给每个 token 做淡入、位移或模糊动画。
- 正文高度随原生内容自然增长。
- 生成中状态只使用一个低强度状态点、光标或边界变化。
- 停止生成按钮必须始终比环境动画醒目。

### 10.4 编辑、删除与滑动

- 编辑开始：当前消息边界在 `125ms` 内增强，不移动正文。
- 保存编辑：允许 `360ms` 的一次性 selection tint 淡出。
- 删除：服从原生立即删除；不拦截数据操作等待离场动画。
- Swipe：不覆盖原生方向、按钮和计数；Rhapsody 只美化状态。

### 10.5 消息操作按钮

- 鼠标场景可在消息 hover 时提高可见度。
- `focus-within` 时必须保持可见，不能依赖鼠标。
- 操作按钮出现不改变消息正文宽度。
- 长消息滚动过程中不反复触发 hover 动画。

## 11. 输入区

输入区是最高频交互，应比其他区域更克制。

### 11.1 聚焦

- `#send_form:focus-within` 在下一帧显示强边界。
- `180ms` 内建立低强度光晕。
- 不改变输入区尺寸和底部位置。

### 11.2 输入

- 键入、删除、IME 组合输入和粘贴不附加动画。
- 不监听每次输入去驱动 OGL 或背景效果。
- 多行增长保持原生速度，不对 height 做 transition。

### 11.3 发送

```text
点击 / Enter
→ 立即显示 press
→ 原生发送逻辑执行
→ 用户消息进入
→ 发送按钮切换为原生生成 / 停止状态
```

- 发送按钮不播放长波纹。
- 可使用一次 `160ms` 的局部高光扩散，但不得覆盖输入文字。
- 断开连接、不可发送和生成中状态服从原生 class。

### 11.4 附件

- 文件附件出现时作为整体 `200ms` 淡入。
- 删除按钮立即响应。
- 文件名截断不能因 hover 改变布局。

## 12. 设置中心

设置中心采用 D 方案的紧凑结构。

- 分类切换时，字段内容使用 `160–220ms` 的短交叉淡入，不横向飞页。
- 字段值先更新 UI，再防抖保存。
- 右侧视觉预览最多以显示器刷新率更新，但参数输入需要在同一 RAF 合并。
- 拖动环境参数时不重建 Renderer、Program 或 Geometry。
- “重置”必须二次确认，并明确只重置 Rhapsody。
- 设置保存成功不连续弹 toast；只有失败或需要重载时提示。

## 13. 主题切换

主题切换只替换语义变量和环境参数，不更换 DOM。

```text
用户选择主题
→ 同帧更新选中状态
→ 写入 data-rhapsody-theme
→ 360–420ms 过渡主要表面与文字
→ 800–1200ms 让 OGL 光色缓慢收束
→ 清除 transitioning 状态
```

规则：

- 禁止使用 `* { transition: ... }`。
- 只给已知表面、边界、文字、图标和阴影定义 transition。
- 不过渡布局尺寸、滚动位置和字体族。
- 大面积背景先改变底色，环境光场随后追随，避免白闪或黑闪。
- 快速重复切换时取消旧动画，从当前计算颜色过渡到最终主题。
- 切换期间原生操作保持可用。

## 14. 诗性时刻

角色开场白和章节过渡可以短暂使用 `Poetic/Display` 与 `Poetic/Quote`。

- 一次只出现一个诗性标题。
- 位移不超过 `12px`，时长 `640–800ms`。
- 允许标题、短引文、细线标记三层错峰，间隔 `60–90ms`。
- 正文开始后立即回到 Noto Sans SC。
- 不在每条角色消息前重复章节动画。
- 用户滚动或开始输入时，叙事动画直接收束到最终状态。

## 15. 环境与指针交互

### 15.1 视差

- 最大视觉位移 `6px`。
- 指针事件只记录目标坐标，不直接写 DOM 或调用 Renderer。
- 在统一 RAF 中使用与 `deltaTime` 相关的阻尼插值。
- 指针离开窗口后在 `800–1200ms` 内回到中性位置。
- 不改变正文、按钮或输入框的位置。

推荐插值：

```js
const alpha = 1 - Math.exp(-deltaTime / smoothingMs);
current += (target - current) * alpha;
```

### 15.2 Dot Field

Dot Field 只能用于欢迎空白区、设置预览或低信息密度背景。

- 不覆盖长聊天正文。
- Pointer glow 与 bulge 在同一环境循环中更新。
- Bulge 不影响真实 DOM 命中区。
- 离开作用区域后平滑归零。
- 不与另一个全屏粒子系统建立第二条 RAF 循环。

### 15.3 OGL Flowmap

- 只用于角色立绘或远景光场，不作为按钮反馈。
- 指针速度影响应经过限幅，快速甩动不会产生高亮爆闪。
- 角色切换时复用 Renderer，替换纹理或参数，不重复创建 Canvas。

## 16. 序曲衔接

- SillyTavern `APP_READY` 后才显示序曲。
- 点击“进入酒馆”后，序曲在 `360–520ms` 内完成最后收束并移除 root。
- 酒馆工作区在序曲后半段已处于最终布局，不再进行一次大幅二次入场。
- 序曲被关闭或本会话已显示时，工作区直接可用，不播放替代假加载。
- 具体碎片与意象时序以 `MOTION-SPEC.md` 为准。

## 17. 减少与关闭动态效果

### Full

执行本规范的完整交互、叙事和环境运动。

### Reduced

- 功能反馈保留，但位移改为 opacity / color。
- 抽屉与 popup 时长不超过 `160ms`。
- 消息不位移，仅轻微淡入。
- 欢迎和章节直接显示最终排版或使用不超过 `200ms` 的淡入。
- OGL 切换到静态 CSS 光场。
- 序曲直接显示最终意象、标题和进入按钮。

### Off

- 所有非必要动画时长为 `0ms`。
- 保留 focus、selected、disabled、error 等静态状态。
- 环境层静态或关闭。

默认偏好为 `system`：系统请求减少动态时计算为 Reduced，否则为 Full。用户显式选择优先于系统的非强制外观设置，但界面需要清楚说明影响。

## 18. 键盘与焦点

- Tab 顺序保持 SillyTavern 原生 DOM 顺序。
- Enter / Space 继续触发原生按钮和开关语义。
- Escape 关闭最上层 popup、drawer 或序曲。
- Rhapsody 不把纯装饰元素加入 Tab 顺序。
- 环境 Canvas 永远不可聚焦。
- Popup 关闭后焦点返回触发器。
- 动画过程中不能把焦点留在 `visibility: hidden` 或已移除节点。

## 19. 实现技术选择

### CSS Transitions / Animations

用于 hover、focus、press、颜色、消息单次进入和简单面板过渡。

### Web Animations API

用于需要取消、反向或从当前状态继续的主题、drawer 与诗性时刻。Animation 实例必须保存在插件状态中并在销毁时取消。

### requestAnimationFrame

只用于 OGL、Dot Field、Flowmap、指针阻尼和必须逐帧计算的视觉。所有程序化效果共享一个调度器。

### 禁止模式

- `setInterval(..., 16)` 驱动动画；
- 动画 width、height、top、left 形成持续布局；
- 在 pointermove 中同步读取布局并写样式；
- 为每条消息创建永久 RAF、Observer 或事件监听器；
- 用 React 重渲染 SillyTavern 原生消息树。

## 20. 验收标准

- 点击、hover、focus 和输入反馈无可感知等待。
- 60Hz 显示器上日常动画稳定贴近 60 fps；高刷新率显示器不被人为限制为 30/60 fps。
- 抽屉、主题、消息和环境动画同时发生时，输入与滚动仍流畅。
